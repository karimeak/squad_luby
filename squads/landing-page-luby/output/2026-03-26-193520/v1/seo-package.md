## Brief de Performance para Lucas

### Targets obrigatórios (Core Web Vitals)

- **LCP < 2.5s** — Largest Contentful Paint: hero image deve carregar rápido
- **INP < 200ms** — Interaction to Next Paint: formulário e CTAs devem responder imediatamente
- **CLS < 0.1** — Cumulative Layout Shift: todas as imagens com width + height explícitos

### Implementações obrigatórias

**Hero Image:**
```html
<link rel="preload" as="image" href="/images/hero-staff-aug.webp" />
```
- Formato: WebP (fallback AVIF opcional)
- Tamanho máximo: 400KB
- Dimensões: 1440×800px (desktop), gerar versões para mobile (375px, 768px)

**Imagens below-fold:**
```html
<img src="..." alt="..." width="..." height="..." loading="lazy" />
```
- NUNCA omitir `width` e `height` — causa CLS
- Todas em WebP

**Google Fonts Inter:**
```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;600;700&display=swap" rel="stylesheet" />
```
CSS: `font-display: swap` obrigatório (já incluído no URL acima via `display=swap`)

**Google Tag Manager:**
```html
<!-- GTM no <head> — substituir GTM-XXXXXXX pelo ID real -->
<script>(function(w,d,s,l,i){...})(window,document,'script','dataLayer','GTM-XXXXXXX');</script>
```

**Scripts externos:**
```html
<script src="..." defer></script>
```
Nenhum script externo não-crítico sem `defer`. Zero render-blocking scripts no `<head>` além do GTM.

**CSS:**
- Sem stylesheets externas bloqueantes. Tailwind inline ou CSS modules apenas.
- Critical CSS inline no `<head>` para o que está above-the-fold.

**Canonical:**
```html
<link rel="canonical" href="https://luby.co/staff-augmentation" />
```

**Schema JSON-LD:**
```html
<script type="application/ld+json">
  [inserir o JSON-LD completo acima]
</script>
```
⚠️ IMPORTANTE: Schema deve ser inserido como `<script type="application/ld+json">` no `<head>`, NÃO via JavaScript dinâmico. Isso garante indexação correta pelo Google.

**Formulário:**
- Formulário de 5 campos deve ser carregado com HTML estático (não hidden + JS reveal)
- Campos devem ter `autocomplete` attributes corretos (name, organization, email, etc.)
- Submit via AJAX ou fetch — não page reload — mas form HTML deve existir no DOM
